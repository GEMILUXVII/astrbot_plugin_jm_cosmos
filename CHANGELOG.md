## **v2.7.2** (2026-06-18)

### Bug 修复
- **修复 `client_type=html` 时 `/jmfav` 查看收藏失败**
  - HTML 客户端的 `favorite_folder` 需要 `username` 来拼接 `/user/<name>/favorite/albums`，而插件每次新建的 client 不带 `_username`，导致抛“favorite_folder方法需要传username参数”
  - 现在把当前登录用户名透传给 `favorite_folder`（API 端忽略该参数、HTML 端使用），默认 api 模式不受影响

### 新增功能
- **取消收藏** - 新增 `/jmfav del <本子ID>`
  - `add`/`del` 共用幂等逻辑：先用本子详情的 `is_favorite` 判断当前状态，仅当与目标不一致时才切换，避免 toggle 接口被反向操作

### 改进
- **配额模块清理** - 移除无调用方的 `check_quota`/`consume_quota`/`get_remaining`（已由 `reserve`/`refund` 取代）；插件初始化时调用 `cleanup_old_data()`，避免 `quota.db` 随时间无限增长

### 文档
- 帮助信息与收藏提示补充 `/jmfav del` 用法
- 更新版本号与更新时间

## **v2.7.1** (2026-06-18)

### Bug 修复
- **修复 `/jmfav add` 收藏失败、且提示为空的问题**
  - 根因：移动端 API 登录得到的会话（`AVS`）只对 API 端有效，对网页（HTML）端无效；旧实现改用 HTML 客户端的 `/ajax/favorite_album`，被服务端判为“未登录”，返回 `{"status":0,"msg":""}`，最终显示为空白的“收藏失败:”
  - 改为复用登录所用的 API 客户端，直接 `POST /favorite`（jmcomic 的 `JmApiClient.add_favorite_album` 以 GET 请求 `/favorite`，等同“列收藏”且响应无 `status` 字段，会触发 `KeyError('status')`，故绕开该方法）
  - 收藏失败时回传服务端真实原因，不再出现空白错误提示

### 改进
- **`/jmfav add` 改为幂等操作**
  - `/favorite` 是“切换（toggle）”接口，对已收藏的本子再次调用会将其取消
  - 现在先用本子详情的 `is_favorite` 字段判断：已收藏直接提示“已在收藏夹中”，仅未收藏时才执行添加，避免“点添加反而取消”

### 文档
- 更新版本号与更新时间

## **v2.7.0** (2026-06-17)

### 新增功能
- **本子更新订阅** - 新增 `/jmsub`、`/jmunsub`、`/jmsublist` 命令
  - 订阅后由后台按 `subscribe_check_interval` 间隔检查更新
  - 发现新章节时主动推送通知到对应会话
- **增量下载** - 新增 `/jmupdate <ID>`，仅下载已知章节之后的新增章节
- **搜索类型前缀** - `/jms` 支持 `tag:`、`author:`、`actor:`、`work:` 前缀指定搜索类型
- **排行榜分类** - `/jmrank` 支持附加分类参数，如 `/jmrank week hanman`
- **收藏本子** - 新增 `/jmfav add <ID>`，将本子加入收藏夹
- **长图打包** - 新增 `long_img` 打包格式，纵向拼接页面为长图（过长自动分段并打包为 ZIP）
- **下载进度反馈** - 下载较大本子时按 25% 步进推送进度（`show_download_progress` 可关闭）
- **自定义域名 / 重试次数** - 新增 `client_domain`、`retry_times` 配置项，便于绕过被墙的默认域名

### 改进
- **错误信息分类** - 将 jmcomic 异常（本子不存在 / 重试全部失败 / 部分下载失败）映射为可读提示
- **部分下载检测** - 下载后检查下载器成功/失败统计，残缺下载会明确提示而非按完整处理
- **会话持久化** - 登录后保存真实 cookies，重启后自动恢复登录，无需重新登录

### Bug 修复
- 移除 `DownloadResult.cover_path` 被错误赋值为目录的死代码

### 安全与健壮性修复
- **并发安全**：各操作与下载改用独立的 jmcomic client（`new_jm_client`），避免共享 client 在并发线程下竞态；登录后通过 `update_cookies` 注入会话，且不再缓存已认证 client（`get_client()` 每次新建）
- **凭据保护**：`/jmlogin` 在群聊中被拒绝，避免账号密码泄露到群消息/历史/日志
- **配额原子化**：下载前原子预留配额并在失败时返还，修复并发 TOCTOU 超发；管理员不再被计入配额；数据库异常时 fail-open（放行）并显式告警
- **加密失败关闭**：设置了打包密码但缺少 `pyzipper` 时直接报错，不再静默产出未加密 ZIP
- **错误归类**：搜索/详情/章节/排行/推荐/收藏的网络错误统一向上传播，由 `classify_exception` 区分网络失败与本子不存在，不再被吞为“未找到”或“暂无数据”（章节越界仍提示章节不存在）
- **增量完整性**：增量下载（`/jmupdate`）不再因章节过滤被误报为部分失败
- **订阅开关**：`subscribe_check_interval=0` 时不再启动后台任务
- `pack_format=none` 明确为“本地保存不发送”，结果消息提示已保存到本地（不暴露服务器绝对路径）

- 更新 README 命令列表、配置说明与功能特性
- 更新版本号与更新时间
